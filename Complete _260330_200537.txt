## 1. PURPOSE

This document is the single source of truth
for every image asset in GluMira™ V7.

All images have been removed from all code.
No image file is referenced in any script
until it appears in this registry as CONFIRMED.

Every image is referenced by its Asset ID only.
The Asset ID is what appears in all code.
The actual file is dropped into the correct folder
when it is ready.

### 4.2 Logo Design Specification — LOCKED
The GluMira™ logo must contain exactly these elements:
- A blue owl
- Spread wings — not folded
- A wave base beneath the owl's feet
- An amber glucose teardrop — the core data drop
- The wordmark GluMira™ in Plus Jakarta Sans Bold
- Optional submark — powered by IOB Hunter™
  in Inter Regular at 40% scale of main wordmark

Colour rules:
- Owl body: GluMira Navy #0D2149
- Wing detail: Dawn Blue #2D6DB4
- Wave base: Glucose Teal #2A8FA8 at 60% opacity
- Glucose teardrop: Golden Drop #D97706
- Wordmark: GluMira Navy #0D2149 on light
  Pure White #FFFFFF on dark

The owl must never:
- Look aggressive or predatory
- Have sharp angular features
- Lose the wave base in formal use
- Appear without the amber teardrop
- Be placed on a teal background

### 5.2 Mira Character Design Specification — LOCKED
Mira is a barn owl — not a generic owl.
Her design must be consistent across every asset.
Once the first approved Mira image is generated
the style seed or reference must be locked
and used for every subsequent Mira image.

Mira's fixed design elements:
- Species: Barn owl
- Expression: Always warm, always approachable
- Eyes: Large, round, amber — the same colour
  as the glucose teardrop
- Body: Navy blue primary feathers
- Wing tips: Dawn Blue gradient
- Chest: Soft white or pale cream
- Wave detail: Subtle wave pattern on base
  or feet area — connecting to the logo
- Size in frame: Always friendly scale —
  never looming or intimidating
- Style: Semi-illustrated — not photorealistic,
  not cartoon — sits between the two

What Mira is never:
- Scary, dark, or aggressive looking
- Overly cute to the point of not being credible
- Inconsistent between assets
- Placed against a teal background
- Shown with any medical devices on her body

## 14. CANVA PRO WORKFLOW

### Step 1 — Brand Kit Setup
In Canva Pro create the GluMira™ Brand Kit:
- Upload confirmed logo files
- Set brand colours — all HEX codes from
  01_GluMira-Brand-Standards-V7.md
- Set brand fonts — Plus Jakarta Sans and Inter
- Save as GluMira V7 Brand Kit

### Step 2 — Character Consistency
For Mira character assets:
- Generate first approved Mira in Canva or Midjourney
- Save the style reference or seed
- All subsequent Mira images use the same reference
- Never generate a new Mira without the reference

### Step 3 — Template Library
Create one master template per asset type
in Canva Pro Brand Templates.
Every post, badge, and banner starts from
the master template — never from a blank canvas.

### Step 4 — File Naming Convention
Every exported file named exactly as the Asset ID:
IMG_LOGO_LIGHT.svg
IMG_LOGO_LIGHT.png
IMG_MIRA_NEUTRAL.png
etc.

### Step 5 — Filing
Every confirmed asset filed immediately to
GluMira-V7/07_BRAND/Assets/[correct subfolder]
Status in this registry updated to CONFIRMED
with the date of confirmation.
